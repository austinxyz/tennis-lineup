### 2. 排阵算法内存优化
**提交：** `0785c6b`（含于 feat: add export/import）

**背景：** Fly.io 生产环境频繁 OOM，尤其在生成排阵时。

**根因：** `LineupGenerationService.enumerateSubsets()` 把所有 C(n,8) 组合全存入内存再排序。30人球队 = 580万个子集 ≈ 750MB+，远超 256MB 限制。

**修复：**
- 限制可选球员池为 UTR 最高的 18 人（`MAX_OPTIONAL_POOL = 18`）
  - C(30,8) = 580万 → C(18,8) = 43,758，减少 **99.2%**
- 排序后只保留 top-150 子集（`MAX_SUBSETS = 150`）
- `rawCandidates` 上限 300（`MAX_RAW_CANDIDATES = 300`）
- 添加 JVM 参数：`-Xmx384m -XX:+UseG1GC -XX:MaxGCPauseMillis=200`
