# Mobile UI 优化设计

**日期：** 2026-04-19
**目标：** 将现有桌面优先的 UI 适配到 Mobile（< 1024px），提供良好的触屏体验。

## 背景

现有前端按桌面两栏/三栏布局设计：
- 左侧全局导航 `NavSidebar`（固定 224px）
- 队伍管理页还有第二列 `TeamListPanel`（固定 256px），与详情区并列
- 已保存排阵和排阵生成结果使用 `grid-cols-2`（桌面双列）

在手机窗口下：
- 固定侧栏占据过半屏幕
- 两栏布局横向挤压，内容难读
- 多列卡片被压缩变形

## 目标

1. **侧边栏可隐藏** — 小屏默认折叠，汉堡菜单打开
2. **两栏结构可切换单栏** — 队伍管理页：列表和详情二选一显示
3. **多列布局可降级单列** — 已保存排阵、排阵生成结果在小屏单列堆叠
4. **球员/排阵信息密度友好** — 每行突出姓名/性别/UTR/实际UTR，可展开细节

## 非目标

- PWA / 离线 / 推送通知
- 手势操作（滑动返回等）
- 桌面端重构（桌面布局保持现状）
- 独立 Mobile App（React Native / Flutter）

## 断点

使用 Tailwind 默认 `lg` 断点：

- `< 1024px` → Mobile 布局
- `>= 1024px` → 桌面布局（现状不变）

## 全局交互

### 汉堡菜单 ☰
- 位置：所有页面顶部栏最左
- 行为：点击打开左侧抽屉（`translate-x-0`），遮罩 `bg-black/40` 覆盖右侧
- 关闭：点击遮罩 / 抽屉内点击导航项 / Esc 键
- 桌面（≥1024px）不显示汉堡，侧栏常驻

### 返回按钮 ← 队伍
- 位置：汉堡菜单右侧，仅在子页面显示（队伍详情、已保存排阵、排阵生成、对手分析）
- 行为：回到上一级（队伍列表/首页）
- 实现：Vue Router `router.back()` 或显式 `router.push('/')`
- 桌面不显示（桌面有常驻导航）

### 顶部栏结构

```
┌────────────────────────────────────────┐
│ ☰  [← 队伍]  页面标题         [操作] │
└────────────────────────────────────────┘
```

- 左侧：汉堡 + 可选的返回按钮
- 中间：当前页面标题（粗体，截断溢出）
- 右侧：上下文相关操作（如"重新生成""导出""添加"）

## 页面布局

### 1. 队伍管理

**桌面：** `NavSidebar | TeamListPanel | TeamDetail`（三栏）

**Mobile 两状态：**

- **状态 A：未选队伍**（URL = `/`）
  - 顶部栏：`☰ | 队伍列表 | [+ 新建]`
  - 内容：全宽队伍列表（`TeamListPanel`）
  - 点击某队伍 → 跳转 `/teams/:id`

- **状态 B：已选队伍**（URL = `/teams/:id`）
  - 顶部栏：`☰ | ← 队伍 | 浙江队`
  - 内容：全宽队伍详情（`TeamDetail`）
  - 点击返回 → 回到 `/`

### 2. 队伍详情页 — 球员列表

**Mobile 每行：**
```
┌───────────────────────────────────────┐
│ [性别] 姓名      UTR ●   实:UTR    ▸ │
└───────────────────────────────────────┘
```

- `[性别]`：粉色 `女` 或蓝色 `男` 小徽章
- `姓名`：加粗，13px
- `UTR`：灰色，带绿色圆点表示 Verified
- `实:UTR`：橙色，仅在 `actualUtr != null` 且不等于 `utr` 时显示
- `▸/▾`：点击展开/收起

**展开后**：
- Verified Doubles UTR（若有）
- 性别（全称）
- Notes（白色背景框）
- `[编辑] [删除]` 按钮

**桌面不变** — 仍用现有表格样式。

### 3. 已保存排阵

**Mobile：** 单列（`grid-cols-1`），每张卡片独占一行。

**卡片结构：**
```
┌─────────────────────────────────┐
│ ⭐首选  排阵名称     总: 40.15  │
│                      实: 42.48  │
│ 备注文字（可选，2行）            │
│ ──────────────────────────────  │
│ D1  [女] May Wang  5.05  实:6.80│
│     [男] Orien Xu  6.18  实:6.18│
│ D2  [男] Bin Feng  6.28  实:6.80│
│     [女] Chen Z   4.28  实:4.60│
│ D3  ...                         │
│ D4  ...                         │
│ ──────────────────────────────  │
│ [合法]  2026/04/18              │
│         [↑] [↓] [编辑] [删除]   │
└─────────────────────────────────┘
```

- 头部：⭐ 首选徽章（第一条）+ 名称 + 右上角 UTR 总/实
- 每对占 2 行（每名球员 1 行）：`[性别徽章] 姓名 UTR 实际UTR`
- 底部：合法/失效 + 日期 + 操作按钮

**桌面：** `lg:grid-cols-2`（现状）

### 4. 排阵生成

#### 输入页
```
┌────────────────────────────────┐
│ ☰ | 排阵生成  [重新生成]       │
├────────────────────────────────┤
│ 队伍: [浙江队 ▾]               │
│ 策略: [均衡|激进|自定义]       │
│                                │
│ ▾ 📌 固定位置  [2]             │
│   May Wang       D1            │
│   Orien Xu       D1            │
│                                │
│ ▾ ✓ 包含球员   [3]             │
│   [Bin Feng] [Wei Jing]        │
│                                │
│ ▾ ✗ 排除球员   [1]             │
│   [Bing Cai]                   │
│                                │
│ [   生成排阵   ]                │
└────────────────────────────────┘
```

- 高级选项（固定位置/包含/排除）默认折叠，带徽章显示已选数量
- 大号「生成排阵」主按钮
- 桌面：保留现有多列布局

#### 结果页（Mobile 单列）

每个方案独占一行，卡片样式同已保存排阵，但：
- 顶部显示 `方案 N` + 第一条带 `⭐ 最佳` 徽章
- 卡片内：姓名/性别/UTR/实际UTR（每人一行）
- 卡片底部按钮：`[保留此排阵]` 蓝色主按钮 + `[调整配对 ▾]`

### 5. 对手分析

**只通过下拉选择**（不支持文本输入）：

```
┌────────────────────────────────┐
│ ☰ | 对手分析                   │
├────────────────────────────────┤
│ · 我 方 ·                      │
│ 队伍: [浙江队 ▾]               │
│ 排阵: [⭐ 隐藏主力阵容 ▾]      │
│ ┌ 我方预览 ────────────────┐   │
│ │ D1 [女]May / [男]Orien   │   │
│ │ D2 [男]Bin / [女]Chen    │   │
│ │ ...                      │   │
│ └──────────────────────────┘   │
│                                │
│ · 对 手 ·                      │
│ 队伍: [上海队 ▾]               │
│ 排阵: [主力阵容 ▾]             │
│ ┌ 对手预览 (红色主题) ─────┐   │
│ │ D1 [男]王大 / [男]李二   │   │
│ │ ...                      │   │
│ └──────────────────────────┘   │
│                                │
│ [  🔍 开始分析  ]              │
└────────────────────────────────┘
```

- 4 个下拉：我方队伍 / 我方排阵 / 对手队伍 / 对手排阵
- 选完后即时显示两边预览（绿色白/红色背景区分）
- 对手必须是系统内已存在的队伍且有已保存排阵；若所选对手队伍没有任何排阵，"对手排阵"下拉显示"该队伍暂无排阵，请先添加"

#### 结果页
- 顶部整体胜率大卡（绿色渐变）
- 每线卡片：`D1 vs D1` 左右对战布局，胜率百分比 + 进度条
- 风险线（胜率 < 50%）橙色边框 + ⚠️ 警示 + 黄色建议框
- 底部 AI 点评（蓝色背景）

## 组件设计

### 新增：`AppHeader.vue`
- 统一顶部栏组件
- Props: `title`, `backTo` (可选), `backLabel`（默认"队伍"）
- Slots: `actions` (右侧操作区)
- 包含：汉堡菜单按钮（触发全局 `sidebarOpen` 状态）+ 返回按钮 + 标题 + 操作 slot

### 新增：`SidebarDrawer.vue`
- 包装 `NavSidebar`，加上抽屉遮罩和滑入动画
- 桌面：`lg:static lg:translate-x-0`
- Mobile：`fixed inset-y-0 left-0` + `translate-x-0/-translate-x-full` 切换

### 修改：`MainLayout.vue`
- 引入全局 `sidebarOpen` ref（或 provide/inject）
- Mobile 头部区显示 `AppHeader`，桌面隐藏（头部由各页自己渲染）

### 修改：`TeamManagerView.vue`
- 路由保持嵌套（`/` = 列表，`/teams/:id` = 详情）
- 桌面：列表 + 详情并排（`flex` 布局不变）
- Mobile：二选一显示（根据 `route.params.id` 判断）

### 修改：`LineupHistoryView.vue`
- 当前已有 `grid-cols-1 lg:grid-cols-2`，继续用
- `LineupCard` 内的球员行布局改为"姓名突出 + 性别徽章 + UTR/实际UTR"

### 修改：`LineupCard.vue`
- 球员行结构（每对 2 行）：
  ```
  [gender-tag] {name}  {utr} 实:{actualUtr}
  ```
- Flexbox 子元素：`gender-tag` 定宽，`name` 加粗 `flex:1`，UTR 靠右

### 修改：`TeamDetail.vue`
- 桌面：保留表格（现状）
- Mobile（`lg:hidden`）：新增卡片列表视图，每行展开/折叠

### 修改：`LineupGenerator.vue`
- 表单区 Mobile 垂直堆叠
- 高级选项（pin/include/exclude）改用 `<details>` 折叠
- 结果区 `grid-cols-1 lg:grid-cols-2`

### 修改：`OpponentAnalysis.vue`
- 移除对手文本输入
- 新增对手队伍下拉 + 对手排阵下拉
- 选定后显示两边预览（即时）
- 结果布局：整体胜率卡 + 每线对战卡 + AI 点评

## 数据与 API

### 无后端改动
所有数据（球员、排阵、队伍）已经是按队伍存储的。对手分析的对手阵容来自 `team.lineups`，现有 API 已经支持（`GET /api/teams/{id}/lineups`）。

### 状态管理
- 全局 `sidebarOpen` ref（MainLayout 级别 `provide`）
- 各页面局部展开/折叠用各自的 ref（无需 Pinia）

## 样式约定

- 只用 Tailwind 工具类
- 断点：`sm` (640px), `md` (768px), `lg` (1024px), `xl` (1280px)
- 本次改动只在 `< lg` 生效
- 颜色主题：
  - 女：`bg-pink-100 text-pink-700`
  - 男：`bg-blue-100 text-blue-700`
  - UTR：`text-gray-500`
  - 实际 UTR：`text-amber-500 font-semibold`
  - Verified 点：`bg-emerald-500`
  - 首选徽章：`bg-yellow-100 text-yellow-700`
  - 成功/合法：`bg-emerald-100 text-emerald-700`
  - 失败/风险：`bg-red-50 text-red-700`

## 迁移与回滚

- 完全基于现有组件和路由扩展，无破坏性变更
- 桌面布局在任何断点下保持现有视觉
- 一次性 PR 可合并；如发现问题，可通过 feature flag 或 git revert 回滚

## 实现顺序（建议）

1. `AppHeader.vue` + `SidebarDrawer.vue` + `MainLayout.vue` 全局框架
2. `TeamManagerView.vue` + `TeamDetail.vue` 单栏切换
3. `LineupCard.vue` 球员行重构（影响三个页面）
4. `LineupHistoryView.vue` 已经单列，验证即可
5. `LineupGenerator.vue` 表单折叠 + 结果单列

## 开放问题

- **PWA 支持**（添加到主屏）：后续单独评估，本次不做
- **iPad 断点**（768-1024px）：目前归入 "mobile"；桌面样式在 ≥1024px 生效。如用户反馈 iPad 上希望用桌面样式，可调整到 `md` 断点
- **字段密度**：已保存排阵卡片信息较多，mobile 下显示高度较大；必要时可默认折叠 D3/D4 显示摘要，点击展开
