## Context

当前系统是一个空项目，需要从零构建网球排阵管理功能。组队管理是整个系统的基础功能，为后续的排阵生成和对手分析提供数据支撑。

**当前状态**：
- 后端：无任何代码
- 前端：无任何代码
- 数据存储：无任何数据

**约束条件**：
- MVP 阶段不使用数据库，采用 JSON 文件存储
- 参考 finance 项目的布局模式（侧边栏 + 主内容区）
- 使用 Vue 3 Composition API 和 Tailwind CSS
- 使用 Spring Boot 作为后端框架

## Goals / Non-Goals

**Goals:**
- 实现完整的队伍 CRUD 操作（创建、查看、更新、删除）
- 实现球员的增删改查功能
- 支持 CSV 和 JSON 格式的批量导入
- 提供线程安全的 JSON 文件存储服务
- 实现侧边栏导航和队伍列表/详情页面

**Non-Goals:**
- 此阶段不涉及排阵生成功能
- 此阶段不涉及对手分析功能
- 此阶段不涉及 AI 调用
- 此阶段不涉及用户认证授权

## Decisions

### 1. JSON 文件存储而非数据库

**决策**：使用 JSON 文件存储队伍和球员数据

**理由**：
- MVP 阶段简化部署，无需额外数据库服务
- 数据量小（队伍和球员），JSON 文件足够
- 便于备份和迁移

**考虑的替代方案**：
- 使用 H2 嵌入式数据库：需要额外依赖和配置
- 使用 MySQL/PostgreSQL：需要外部数据库服务，增加部署复杂度

### 2. ReadWriteLock 实现线程安全

**决策**：使用 Java 的 ReadWriteLock 实现 JSON 文件的线程安全读写

**理由**：
- 支持多读单写，提高并发性能
- 满足"支持 10+ 同时用户"的性能要求
- 原子性写入：先写临时文件再重命名

**考虑的替代方案**：
- 使用 synchronized 块：性能较差，不能区分读写
- 使用数据库锁：此阶段不使用数据库

### 3. 三栏布局：导航侧边栏 + 队伍列表面板 + 详情面板

**决策**：采用三栏布局，参考 finance 项目导航风格

- 左一栏（NavSidebar, w-56）：功能导航，"队伍管理"为当前唯一入口，后续可扩展
- 左二栏（TeamListPanel, w-64）：队伍列表，含"创建队伍"和"批量导入"操作按钮
- 右栏（flex-1）：队伍详情面板，嵌套路由控制（空状态 HomeView / 详情 TeamDetail）

**路由结构**：`MainLayout → TeamManagerView → [HomeView | TeamDetail]`

**理由**：
- 用户可同时看到导航、队伍列表和队伍详情，无需页面间跳转
- 参考 finance 项目已验证的导航模式
- 响应式设计：移动端折叠为顶部 header + 抽屉菜单

### 4. 批量导入作为 Modal 集成，而非独立路由页面

**决策**：批量导入功能以 modal 形式嵌入 TeamListPanel，移除 /import 独立路由

**理由**：
- 操作场景与队伍列表紧密相关，无需独立页面
- 与"创建队伍"按钮并排，交互模式一致
- 减少路由层级，简化用户操作路径

### 4. 前端 Composables 分层

**决策**：将 API 调用逻辑封装到 Composables 中

**理由**：
- 逻辑复用，多个页面可以共享
- 便于测试（独立测试 composable 而非组件）
- 符合 Vue 3 最佳实践

## Risks / Trade-offs

### Risk 1: JSON 文件并发写入冲突
**描述**：多个用户同时写入可能导致数据丢失

**缓解措施**：
- 使用 ReadWriteLock 保证线程安全
- 原子性写入（临时文件 + 重命名）
- 定期备份机制

### Risk 2: 批量导入数据验证不充分
**描述**：导入格式错误或数据范围错误可能导致问题

**缓解措施**：
- 详细的字段验证（UTR 范围、性别枚举、字符串长度）
- 逐行验证，跳过无效数据并记录错误
- 返回导入结果摘要（成功数、失败数、错误详情）

### Risk 3: 删除操作误操作
**描述**：用户可能误删除队伍或球员

**缓解措施**：
- 删除前添加确认对话框
- 删除操作通过 DELETE API，前端显示确认提示

### Trade-off 1: JSON 文件 vs 数据库
**选择**：使用 JSON 文件

**优点**：
- 零配置，开箱即用
- 便于数据迁移和备份

**缺点**：
- 不适合大规模数据
- 查询效率不如数据库

**影响**：MVP 阶段可接受，后续可迁移到数据库
